How split tunneling works
Rules tell the VPN client to skip the tunnel for specific IPs, so that traffic is routed by the client's own network stack instead of being sent to the VPN server.
Prerequisite: the client device must already have a direct route to the target (e.g. on the same LAN as 192.168.169.121). If the client is remote and has no independent path to the target, split tunneling alone cannot reach it — a gateway peer on the target LAN is required.

Verify & fix routing (recommended)
Step 1 — Check which interface is carrying the target
Linux ip route get <target-ip> Must NOT show dev wg0 (or whatever your tunnel interface is called).
macOS route -n get <target-ip> The interface: line should be your LAN (typically en0), not utun*. If it shows destination: default with utun*, the macOS GUI app installed a full default route — use Step 2.
Windows route print <target-ip> Look for the lowest-metric matching route. Interface index should point to your LAN adapter, not the WireGuard tunnel.
Step 2 — Force-bypass with a host route (if Step 1 shows the tunnel)
Some clients (notably the macOS GUI app) install a default route on the tunnel interface regardless of AllowedIPs. When that happens, add an explicit host route via your LAN to shadow the tunnel's default. The commands below must be run after the tunnel is active.
Linux sudo ip route add <target-ip>/32 via <lan-gateway> dev <lan-if> Find your LAN gateway + interface with ip route show default (typically 192.168.x.1 dev eth0 or wlan0).
macOS sudo route -n add -host <target-ip> -interface <lan-if> Find your LAN interface with route -n get default while the VPN is off (typically en0 for Wi-Fi, en1 for Ethernet). Re-run this command every time you reconnect the VPN, or add it to a launch script.
Windows route ADD <target-ip> MASK 255.255.255.255 <lan-gateway> METRIC 1 Run as Administrator. Find your LAN gateway with ipconfig (the "Default Gateway" line under your LAN adapter). Use -p to persist across reboots.
Step 3 — Re-verify
Run the Step 1 command again. The interface should now show your LAN adapter instead of the tunnel. Traffic to <target-ip> will now bypass the VPN.
